# 슬래브 도구 확장 로드맵

현재까지 구축된 `/member-check/slab-span` 기능을 토대로, 앞으로 어떤 방향으로
확장할지 정리한 **순서 중심 로드맵**이다. 구체적 일정·기한은 두지 않는다.
새 안건이 생기면 우선 이 파일에 추가해 맥락을 잃지 않도록 하고, 실행 단위로
쪼개지면 `.claude/plans/*.md` 또는 GitHub Issue 로 승격한다.

관련 단기 운영 개선은 [BACKLOG.md](./BACKLOG.md) 참조(본 로드맵은 **기능 확장**
중심).

## 범례

- **★ 핵심** — 슬래브 설계 업무의 핵심 흐름. 이 없으면 반쪽짜리.
- **◆ 부가** — 실무 편의/품질 향상. 있으면 좋음.
- **◇ 탐색** — 수요·타당성 불확실. 도입 전 간단 PoC 권장.

---

## 현재 완성된 토대

- **데이터 연계** — `/db/NODE`, `/db/ELEM`, `/db/STOR`, `/db/FBLA`, `/db/FBLD`, `/db/SECT`
- **패널 탐색** — Union-Find 기반 자동 탐지 (불규칙 평면, 분할 보, 개구부 지원)
- **하중 매칭** — Floor Load 다각형 point-in-polygon, Wu = 1.2D+1.6L 대표값
- **분류(S) 관리** — TYPE(A~E)/THK/X1~X5/Y1~Y5 단위 배근표 + 색상 자동 그룹
- **영속화** — 이름맵/배근표/스냅샷 JSON 저장 + 복원
- **시각화** — SVG pan/zoom, 양방향 하이라이트, 팝오버 세부내역

## Phase 1 — 배근 파싱과 설계 검토 엔진 (가장 자연스러운 확장)

> 이미 입력받고 있는 배근 문자열과 경간·두께를 실제 설계 지표로 전환.
> MIDAS 해석 결과 없이도 얻을 수 있는 값부터 우선 구현.

| # | 우선 | 항목 | 내용 | 근거 |
|---|------|------|------|------|
| 1.1 | ★ | **배근 문자열 파서** | `HD13@200` → `{dia:13, spacing:200, As_per_m}` 변환 유틸. 이후 모든 검토의 입력 | `kds_rc_beam.py` 의 `REBAR_AREA` 재사용 |
| 1.2 | ★ | **처짐 검토** | `short_span / thk` 로 ℓ/h 자동 판정. 경계조건(TYPE A~E)별 허용비 | KDS 14 20 30 표 4.2-1 |
| 1.3 | ★ | **최소·최대 철근비** | ρmin/ρmax + 온도수축 배근 0.002bh 등 | KDS 14 20 50 |
| 1.4 | ◆ | **뚫림전단 검토** | 기둥 주변 2방향 전단. `/db/ELEM` TYPE=COLUMN 위치 활용 | KDS 14 20 22 |
| 1.5 | ◆ | **간단 휨검토 (해석 없이)** | Wu·Lx² 기반 근사 소요 As (양방향 슬래브 계수법) | KDS 14 20 70 부록 |

## Phase 2 — MIDAS 해석 결과 연계

> MIDAS Plate 해석 결과로 Phase 1 의 근사값을 실제값으로 교체.
> 정밀도가 필요한 공정에서 본격 활용.

| # | 우선 | 항목 | 내용 |
|---|------|------|------|
| 2.1 | ★ | **Plate 부재력 조회** | MIDAS Plate element force API → 패널별 Mx/My/Mxy 집계 |
| 2.2 | ★ | **Wood-Armer 휨철근 산정** | 비틀림모멘트 포함 2방향 소요 철근량. 상/하부 분리 |
| 2.3 | ★ | **DCR 자동 산출** | `As_req / As_prov` → 테이블·SVG 에 색상 코딩 (≥1.0 적색) |
| 2.4 | ◆ | **슬래브→보 하중 분배** | 2방향 분배선(45도 or 강성 비율) 기반 Tributary → 기존 `rc-beam` 과 연결 |
| 2.5 | ◆ | **층별 고정하중 집계** | 내진하중/자중 검증용 층 Σ(Wi·Ai) |

## 병렬 — 지속 개선

언제든 필요 시 손댈 수 있는 상시 개선 항목. Phase 와 독립.

- ◆ **성능** — 수천 패널 모델 대응. 패널 탐색 O(n²) 구간 최적화, virtual scrolling
- ◆ **사선 슬래브/캔틸레버 인식** — 현재 X/Y 축 정렬만. `skew_tol_deg` 확장
- ◆ **배근 TYPE별 시각 심볼** — 상부(실선) / 하부(점선) bar 를 SVG 에 오버레이
- ◇ **MIDAS 뷰 캡처 연동** — 특정 패널 위치를 MIDAS 화면에 포커스
- ◇ **색상 접근성** — 분류 색 자동 생성 시 WCAG 대비비 보장

## 의사결정이 필요한 열린 항목

실행 전 사용자/팀과 결정해야 하는 사항. 이 목록이 비어야 해당 Phase 시작 가능.

- [ ] Phase 1 중 **어느 항목부터 착수**할지 (처짐 / 파서 / 플레이트 연계)
- [ ] Plate 부재력을 **어느 MIDAS API** 로 가져올지 (`/post/PLATEFORCES` vs 시공단계 결과)
- [ ] 리포트 **출력 형식 우선순위** (Excel > PDF > CAD 중 어느 것부터)
- [ ] 팀 공유 시 **서버 DB 선택** (현재 로컬 JSON, 후보: Neon Postgres / SQLite)
- [ ] 배근 문자열 **표준 표기법 고정** (`HD13@200` vs `D13@200` vs `HD13@200 CTC` 등)
