# 출력 계약 — 공유 표준

본 문서는 `.claude/agents/reviewers/`, `.claude/agents/authors/` 모든 에이전트의 검토 결과 출력 표준입니다. 각 에이전트 정의는 본 계약을 참조하고 도메인 특화 항목만 추가합니다.

## 1. 표준 5 섹션 구조

```markdown
# {도메인} 검토 결과

## 1. 부재 정보 요약
{수집된 단면/배근/형상 테이블}

## 2. MIDAS 부재력 / 입력 요약
{부재별 설계 부재력 또는 입력 데이터 테이블}

## 3. 설계 검토
### 3.1 휨 검토
| 부재 | Mu (kN·m) | φMn (kN·m) | 비율 | 판정 |
### 3.2 전단 검토
| 부재 | Vu (kN) | φVn (kN) | 비율 | 판정 |
### 3.x {도메인 특화 검토 — 예: 처짐, 뚫림전단, P-M 상관도}

## 4. KDS 상세 조건 검토
| 검토항목 | 기준값 | 설계값 | 판정 |

## 5. 종합 의견
- 부적합 항목 요약
- 구체적인 개선 방안
```

도메인별 가감:
- **rc-beam-reviewer**: 3.1 휨 / 3.2 전단 / 3.3 비틀림(필요시) / 3.4 처짐 / 3.5 균열
- **rc-slab-reviewer**: 3.1 처짐(ℓ/h) / 3.2 최소철근비 / 3.3 근사 휨 / 3.4 뚫림전단
- **rc-wall-reviewer**: 3.1 휨 / 3.2 전단 / 3.3 경계요소
- **seismic-cert-author**: hwpx 산출물 + 검토 보고는 본 5 섹션
- **design-report-author**: PDF/DXF/Excel 산출물 + 본 5 섹션

## 2. 단위 명기 규칙

모든 수치에 단위를 항상 명기:

| 물리량 | 단위 | 비고 |
|--------|------|------|
| 길이 / 치수 | mm | 도면/MIDAS 입력 모두 mm |
| 면적 | mm² | 철근, 단면 |
| 힘 | kN | |
| 모멘트 | kN·m | 점 사용 (kN.m 금지) |
| 응력 / 강도 | MPa | fck, fy, σ |
| 단위중량 | kN/m³ | |
| 분포하중 | kN/m² (면) / kN/m (선) | |

단위 변환 규칙:
- MIDAS 응답이 SI 와 다른 단위라면 **즉시 SI 로 변환** 후 처리
- 단위 변환 사실을 출력 섹션 하단 각주로 명시
- 절대 단위 혼용 금지 (mm + m 동시 사용 X)

## 3. 판정 표기

각 검토 항목 마지막 칼럼:

| 표기 | 의미 | 색상 (UI 표현 시) |
|------|------|---------|
| **O.K** | 기준 만족 | 녹색 |
| **N.G** | 기준 미달 | 적색 |
| **검토 필요** | 정보 부족 또는 경계조건 불확실 | 회색 |
| **참조** | 검토 대상 외 (예: 비계산 항목) | 무색 |

비율(DCR) 표기 보조:
- ≤ 0.80: O.K (여유)
- 0.80 < 비율 ≤ 1.00: O.K
- 1.00 < 비율 ≤ 1.05: N.G (재검토 권장)
- > 1.05: N.G (개선 필수)

## 4. 종합 의견 필수 요건

5번 섹션 "종합 의견" 은 다음을 **반드시** 포함:

1. **N.G 항목 전부 나열** — 부재명 + 항목 + 비율 또는 차이값
2. **각 N.G 항목별 개선 방안** — 구체적 (예: "주근 3-D25 → 4-D25 로 변경", "스터럽 간격 200 → 150 으로 축소")
3. **종합 판정** — 전체 부재의 적정성 (예: "모든 부재 적정", "B3, B7 재설계 필요")

개선 방안 작성 원칙:
- 가능하면 1순위/2순위 대안 제시
- 각 대안의 장단점 1줄
- 시공성/경제성 영향 언급

## 5. 품질 보증 체크리스트

출력 전 자체 검증:

- [ ] 계산 과정이 단계별로 명시되어 추적 가능한가
- [ ] 단위가 모든 수치에 명기되었는가
- [ ] 안전측 설계 원칙을 따랐는가 (불확실 시 안전 측)
- [ ] O.K / N.G 가 명확히 표기되었는가
- [ ] 부적합 항목에 구체적 개선 방안이 있는가
- [ ] KDS 조항 번호가 명시되었는가
- [ ] MIDAS API 응답 단위와 SI 변환 일관성 확인했는가

## 6. 보조 산출물 (authors 전용)

`authors/` 에이전트는 본 5 섹션 검토 보고와 함께 다음 산출물을 생성:

| 에이전트 | 주 산출물 | 부 산출물 |
|---------|----------|-----------|
| seismic-cert-author | hwpx (내진확인서) | 5 섹션 보고 |
| design-report-author | PDF (보고서) + DXF (도면) + Excel (배근표) | 5 섹션 보고 |

산출물 파일명 규칙:
- 공백 → 하이픈 (GitHub Release 호환, 메모리 feedback_release_filename.md 참조)
- 한글 사용 가능, 특수문자 금지
- 형식: `{프로젝트명}_{도메인}_{날짜YYYYMMDD}.{확장자}`

## 7. MIDAS API 연동 표준

코드 예시 (모든 reviewer 가 사용):

```python
import MIDAS_API as MIDAS

MIDAS.MIDAS_API_BASEURL("https://...")
MIDAS.MIDAS_API_KEY("your-api-key")
response = MIDAS.MidasAPI("GET", "/db/ELEM")
```

- 파이썬 코드는 반드시 `.venv` 가상환경에서 실행
- ContextVar 기반 클라이언트 사용 (요청별 상태 분리)
- 자세한 엔드포인트는 `.claude/skills/midas-api/index.md` 참조
