# KDS Code Rule

## 목적

건축구조기준(KDS) 기반 계산 로직을 코드에서 일관되게 유지하고, 재사용 가능한 엔진 모듈로 분리한다.

## 적용 범위

- `backend/engines/**/*.py`
- KDS 계산식을 포함하는 서비스/라우터/유틸 코드

## 핵심 원칙

1. KDS 계산식은 라우터/컨트롤러에 직접 작성하지 않고 `backend/engines` 모듈에 둔다.
2. 규정값(계수, 한계값, 표 기반 값)은 하드코딩 최소화 후 상수/테이블로 분리한다.
3. 계산 입력/출력은 단위와 의미를 명확히 명명한다. (`sds`, `sd1`, `ie`, `r`, `cs` 등)
4. 코드 주석에는 KDS 조항 번호 또는 표 번호를 함께 남긴다.
5. 반올림/보간/상하한 처리 규칙은 함수로 분리해 중복을 피한다.

## 구현 가이드

- 엔진 함수는 "순수 계산 함수" 우선으로 작성하고 I/O 부작용(DB, 파일)을 분리한다.
- API 계층에서는 입력 검증과 예외 매핑만 수행하고 계산은 엔진에 위임한다.
- 결과 객체는 후속 문서화/보고서 생성에서 재사용 가능하도록 구조화한다.

## 금지 사항

- KDS 계산식의 임의 축약 또는 근거 없는 상수 변경
- 동일 계산 로직을 여러 파일에 복사-붙여넣기
- 단위가 섞인 입력값을 변환 없이 직접 계산

## 검증 체크리스트

- [ ] 계산식과 변수가 KDS 조항/표와 매칭되는가
- [ ] 경계조건(최소/최대/예외 케이스) 테스트가 있는가
- [ ] 엔진 모듈이 다른 기능에서도 재사용 가능한가
